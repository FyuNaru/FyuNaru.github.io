实验要求

  • 基于PyTorch实现VGG/ResNet/SENet结构

  • VGG要求Conv部分参照论文，可动态调整结构，测试结果不低于80%； ResNet要求基于残差块，参照论文，可动态调整，测试结果不低于80%； 分别在上述VGG和ResNet基础上，添加SE block。

  • 基于VGG进行训练方式对比（学习率、优化器(SGD与Adam)对比）

  • 基于VGG进行data augmentation对比分析（基于上述最佳模型，讨 论分析翻转、旋转、移位等操作对结果影响）

  • 在Cifar-10数据集上进行验证

实验环境

实验环境如下：

  • Win10
  • python3.8
  • Anaconda3
  • Cuda10.2 + cudnn v7
  • GPU : NVIDIA GeForce MX250

tensorboard的运行方法仍同上一实验：

打开pycharm的terminal，输入tensorboard --logdir result

然后在浏览器打开localhost:6006即可.

本实验的过程主要分以下几部分，将根据实验指导书要求的采分点分块来讲。

该类继承了torch.utils.data.Dataset，手动实现了__getitem__()方法和__len()方法。

在此类实现的过程中，重难点是对数据类型及格式的变换。

首先，从官方下载的数据集文件中直接读入的数据为10000 * 3072大小的二维矩阵，该矩阵的每一行中，前1024为图像的R，中间1024为G，最后1024为B。用reshape函数可将其转化为10000 * 3 * 32 * 32大小的矩阵，我们最终传进网络的数据的维度也应该是这样的（指每幅图像以3 * 32 * 32的形式传入）

然而，若想对数据进行数据增强操作，其多种数据增强函数所要求的数据组织形式与数据格式是不同的，例如使用图像翻转函数时，要求为32 * 32 * 3，且为图像格式（使用opencv或PIL实现，我这里使用的是PIL），而使用BN时，又要求传入3 * 32 * 32的Tensor格式。因此需要进行一些额外的数据格式的转换操作。具体的转换操作详见代码。

最终实现的Dataset与原先官方的Dataset功能大致相同，直接读取时返回32 * 32 * 3的ndarray的数据，将其用dataloader包装时，则可以返回经过数据增强的3 * 32 * 32的Tensor格式的数据。

VGG

VGG的基本结构如下，

其主要特点是以多个连续的、较小的卷积核替换一个较大的卷积核，来对AlexNet进行改进，使得感受野相同的情况下，不仅可以使参数更少，同时也增加了层数，可以学习更加复杂的模式。

VGG论文中提出的结构主要有以下几种，我选择的是VGG16，整体结构与论文中的一致，修改了部分参数。（主要是由于训练集图片较小，所以减小了一开始的卷积核大小，和最后的全连接层的节点数量）

ResNet

ResNet的基本结构如下，

其主要特点是，通过引入残差块的方式，消除了网络层数过多时的梯度弥散/爆炸问题，从而可以大大增加网络层数，使得学习结果变好。

我在本实验中选取的是ResNet18，选取该网络的原因是由于数据集过小，经过一定的层数后数据就无法进一步缩小了（在我的网络中经过ResNet18后图片由32 * 32变为了4 * 4，只能再做一些下采样操作，不足以继续增加层数）所以需要选取层数较少的网络。

SE_ResNet

SE block最先在ResNet中使用，其结构如下所示：

SE block并不是一个完整的网络结构，而是一个子结构，可以嵌到其他分类或检测模型中。SENet的核心思想在于通过网络根据loss去学习特征权重，使得有效的feature map权重大，无效或效果小的feature map权重小的方式训练模型达到更好的结果。

VGG_ResNet

原先的VGG网络中，以卷积核大小将所有的卷积核分为了五层，我在这五层外侧都各添加了一个se block，其他部分的结构保持不变。所用的se block与ResNet中使用的一致，只是根据实际情况修改了其中卷积核的大小。

对比试验

实验指导书上要求的对比试验主要有三部分：一是对比更换学习率的影响，二是对比更换优化器的影响，三是对比使用数据增强的影响。

对比学习率部分，我是采用的adam的0.0001和0.0005来对比

对比优化器部分，我采用了SGD和Adam来进行对比。

三是对比使用数据增强的影响，这里我所做的数据增强为随机翻转和归一化。

半精度实现

半精度的实现较为简单，在将数据和模型放置到GPU的语句后加一句.half()即可实现，观察结果发现有较小的速度提升，但差别不大，可能是迭代次数较小的原因。

由于在VGG中加入半精度经常会导致内存报错，为保证稳定性，因此只在ResNet中加入了半精度。

总结：通过比较可知，使用SE对结果的提升效果明显，如下图所示，在30个epoch时SEResNet的结果就可以达到一个很高的值，使用更少的epoch就可以达到同样的效果。使用SE对于ResNet的提升较为明显，但是对于VGG提升不是很明显，原因是由于学习率设置的较大时，会导致过拟合，而过小又会导致结果不是很好，推测是网络结构所致。

关于VGG的对比实验

对比是否使用数据增强时，我统一采取了Adam，且学习率为0.0001的数据，一次使用了数据增强，另一次没有使用数据增强，下面两图分别为没有使用和使用的

使用的epoch都为30，可以看到，使用数据增强，在30个epoch左右即达到了实验要求的80%，比没有使用epoch的高了5%。因此在ResNet中也使用了数据增强，50个epoch时已经达到了很高的结果88%，如下图

未使用数据增强的情况由于学习率不达标，因此没有保存，使用数据增强的模型放在path = "产生模型/2/model_RE_ResNet.pth下

对比学习率时，统一使用adam优化器，学习率分别设为了0.0001和0.0005，

学习率为0.0001的模型在上面已经展示过，学习率为0.0005的模型放在path = "产生模型/6/model_VGG.pth下

学习率为0.0001时，准确率增长较慢，30个epoch的情况下也只达到了76%的识别率，而学习率为0.0005时，如下图所示，仅在15个epoch即可上80%（左侧的折线是由于训练失败时忘记删除log文件，导致多次的结果重合在一起了，从epoch5开始看即可）

对比优化器时，使用SGD会导致过拟合，尝试了多种不同大小的学习率都产生了过拟合。因此可以得出，对于VGG网络来说，Adam更适合一些。
